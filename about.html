Project Information

About / Engineering Log

Lebanon Weather & Travel Explorer is a front-end final project by Charbel Chehade that combines curated Lebanese travel content with current weather lookup tools.

Project Idea

The goal of this website is to help users explore real places in Lebanon and make basic travel decisions before visiting. The site includes a home page, a searchable destinations page, a weather page, and this engineering log.

The project focuses on front-end development requirements: semantic HTML5, hand-written CSS3, Bootstrap 5, Flexbox, vanilla JavaScript, ES6 classes, responsive layout, and public API integration.

API Used

The weather page uses the OpenWeather Current Weather API to request live weather data for Lebanese cities. The interface displays the city name, temperature, condition, humidity, and wind speed.

The private API key is not hardcoded into the project. A clearly marked placeholder is kept in assets/js/weather.js so a tester can add a personal key locally.

Curated Data

The destinations page uses local curated content from assets/js/data.js. The data includes real Lebanese places such as Baalbek, Jeita Grotto, Byblos, Cedars of God, Tyre, Batroun, Beiteddine, Qadisha Valley, Anjar, and Raouche.

Each destination includes structured fields such as region, city, category, short description, best season, estimated visit time, image, tags, coordinates, and featured status.

Custom Requirement

Charbel Chehade's custom requirement is a collapsible sidebar menu on smaller screens. On larger screens, the sidebar remains visible. On smaller screens, it opens behind a menu button, closes smoothly, updates aria-expanded, and closes when a mobile navigation link is clicked.

This requirement is implemented with vanilla JavaScript in assets/js/sidebar.js using an ES6 class.

Main Technical Decisions

  • Bootstrap 5 is used for the responsive navigation, form controls, buttons, cards, grid helpers, and utility classes.
  • Custom CSS is kept in assets/css/styles.css for project-specific layout, sidebar behavior, cards, hero styling, filters, and weather states.
  • Vanilla JavaScript is organized with ES6 classes: SidebarMenu, DestinationExplorer, FeaturedDestinations, WeatherService, and WeatherPage.
  • The destination data is stored locally so the page can render curated content without requiring an external content API.
  • The weather feature separates API calls from UI behavior by using a service class and a page class.
  • The OpenWeather key remains a placeholder to avoid exposing a private secret in front-end source code.
